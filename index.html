Features

Dictation that gets out of your way.

One muscle-memory move: press, speak, release. Everything between is automatic.

Global hold-to-talk

A system-wide hotkey works in any app. Hold fn to talk, or press ⌘+fn for hands-free toggle mode.

LLM post-processing

Raw transcripts are polished by an OpenAI-compatible model — punctuation, capitalization, filler removed — before they're pasted.

Screen-aware context

Active window title, selected text, and an optional ScreenCaptureKit snapshot are sent as context — so the LLM understands what you're talking about.

Groq fallback

Configure a fallback provider (defaults to Groq with openai/gpt-oss-20b) so dictation keeps working when your primary endpoint hiccups.

Local pipeline history

Each dictation is logged on-device with timings, models used, and the cleaned output — replay or copy any past transcript.

Keys in the Keychain

API keys live in macOS Keychain. Audio and prompts go directly from your Mac to the endpoint you configure — never through us.

How it works

The pipeline, end to end.

Four stages between key-down and the cleaned text landing in your focused field.

  1. 01

    Hotkey fires

    Default hold is fn, toggle is ⌘+fn. The recording overlay appears, audio capture begins.

    fn
  2. 02

    Audio captured

    AudioRecorder streams from the selected input device with live level metering, until you release the key.

  3. 03

    Transcribe & clean

    Sent to an OpenAI-compatible transcription endpoint (default model gpt-4o-mini-transcribe), then an LLM polishes it with screen context.

  4. 04

    Paste into focused app

    The cleaned text is dropped into whatever text field is in front of you — Mail, Slack, your editor, Terminal, anything.

    |
Shortcuts

Muscle memory in two keys.

Defaults are out of the way. Rebind anything from Settings if your fingers disagree.

  • fn
    Hold to talk Hold while you speak. Release to transcribe and paste.
  • ⌘+fn
    Tap to toggle Hands-free recording. Tap again to stop and paste.
  • esc
    Cancel Discard the current recording without transcribing.

FAQ

Quick answers.

What macOS version do I need?+

Stro Speak targets macOS 26.2 (Tahoe). It's built against the current SDK, so a recent macOS install is required.

Do I need an API key?+

Yes — you bring your own. Stro Speak talks to any OpenAI-compatible endpoint for transcription and LLM cleanup. Defaults work with OpenAI and Groq; keys are stored in macOS Keychain.

What models does it use by default?+

Transcription: gpt-4o-mini-transcribe. LLM cleanup: gpt-5.4-nano with gpt-5.4-mini as fallback. The Groq fallback provider uses openai/gpt-oss-20b. All configurable in Settings.

What permissions does it need?+

Microphone (to record), Accessibility (to detect the cursor and paste into other apps), and optionally Screen Recording (for ScreenCaptureKit context snapshots). You'll be prompted on first launch.

Where does my audio go?+

Straight from your Mac to whichever endpoint you've configured. Stro Speak doesn't run any servers — audio and prompts never touch our infrastructure.

Is it open source?+

It's a heavily refactored fork of FreeFlow (MIT). Source for the current build lives on GitHub.
